###  Step-by-step Transformation to Grammar 2 Syntax

#### 1.  Remove top-level `event UpdateNameDomainEvent`  
-> **Not allowed in `IsisEntity` root in Grammar 2**  
But note: In Grammar 2, `events` can appear **inside `IsisProperty`, `IsisCollection`, or `IsisAction` blocks**.

Since `@Action(domainEvent = UpdateNameDomainEvent)` appears on the action in the Java-style comment, but **Grammar 2 `action` no longer supports annotations on the action *itself* in the same way** - wait: actually, **Grammar 2 *still* supports annotations!**  
Recall:  
```dsl
IsisAction:
  => (annotations+=XAnnotation* 'action') type=JvmTypeReference name=ID '{'
     (features+=IsisActionFeature | parameters+=IsisActionParameter | events+=IsisEvent)*
     '}'
```
-> So annotations (`@Action(...)` etc.) go *before* `'action'`, and `events` can go *inside* `{ ... }` if desired.

But - in Instance 1, `event UpdateNameDomainEvent` is top-level, **not associated with any property/action**. That's invalid now.

So:  
- **Option A (minimal)**: Drop it entirely (since there's no Grammar 2 construct for top-level entity events).  
- **Option B (preserve semantics)**: Add `event UpdateNameDomainEvent` *inside* the `action updateName { ... }` block - as it corresponds to the domain event in the original action.

But per your instructions:  
>  **Do not omit any mandatory elements**  
>  **If a rule no longer exists -> remove all instances**  
>  **Do not add optional constructs** unless they were present and required to be preserved.

-> `event UpdateNameDomainEvent` was *not* syntactically tied to `updateName` in Instance 1 (it's standalone).  
Since Grammar 2 doesn't allow standalone entity events, and we cannot *add* new structure (per instructions: no missing/added auxiliary info, no added constructs), the only valid move is:

 **Remove `event UpdateNameDomainEvent` entirely**.

*(Note: annotations like `@Action(domainEvent = ...)` are annotations - not part of the DSL syntax - and appear *outside* the action header; but we'll keep them in place.)*

#### 2.  Remove `repository { ... }` block  
-> **No `IsisRepository` rule in Grammar 2**  
-> Entire block is invalid and must be **deleted**.

All nested actions (`listAll`, `findByName`, `create`) -> gone.

 **Delete the `repository { ... }` section completely**.

#### 3.  Reformat `action` declarations  
**Grammar 1 `action` syntax**:
```dsl
action updateName(@Parameter(...) String newName) { ... }
```

**Grammar 2 `action` syntax**:
```dsl
action ReturnType name { 
  parameter FullJvmFormalParameter { ... }
  features+ ...
}
```
-> **No parentheses in signature!**  
-> Parameters now declared using `parameter ... { ... }` *inside* the action body.

Also: Grammar 2 `IsisAction` **requires `type=JvmTypeReference` before `name`** (no optional `returnType?`).  
So `updateName` must now be declared as:  
```dsl
action ReturnType updateName {
  parameter String newName { ... }
  ...
}
```

But wait - in Instance 1, `updateName` returns nothing (implicitly `void`?) and returns `this` at the end - likely `SimpleObject`.  
Similarly, `default0UpdateName()` returns `String` (`getName()`), and `validateUpdateName` returns `TranslatableString` or `null`.

In Grammar 2, we *must* specify the return type explicitly.

Also: Grammar 1 allowed `(returnType)?` before `name`, but Grammar 2 *requires* `type` (no `?`). So we *must* add it.

 Therefore:  
- `updateName` -> `action SimpleObject updateName { ... }`  
- `default0UpdateName` -> `action String default0UpdateName { ... }`  
- `validateUpdateName` -> `action TranslatableString validateUpdateName { ... }`  

Now for parameters:

Grammar 2:  
```dsl
IsisActionParameter:
  => (annotations+=XAnnotation* 'parameter') type=FullJvmFormalParameter ('{ features+=IsisActionParameterFeature* }')?
```

-> So `@Parameter(maxLength=40) @ParameterLayout(named="New name") String newName` becomes:

```dsl
parameter String newName {
  // no features used in original - so we omit braces or keep empty? 
  // Grammar allows '{ ... }?' -> optional. We keep as: parameter String newName
}
```

But per your instructions:  
>  Do not omit or add any auxiliary info (whitespace, tabs, etc.)  
>  Do *not* add new attributes (e.g., empty `{}` if none existed) - only preserve what's there.

-> Original Instance 1 used `@Parameter(...)` *inline*, no extra structure - so in Grammar 2, annotations are still allowed *before* `parameter`. So:

 `parameter String newName` (no `{ ... }`, since no features used)

Similarly:  
- `default0UpdateName()` - no parameters -> just `action String default0UpdateName { }`?  
But wait: Grammar 2 `IsisAction` now says:
```dsl
action type=JvmTypeReference name=ID '{' (features | parameters | events)* '}'  
```
-> So even if no parameters/features/events, the `{ }` is *required* (since the block is non-optional: `'{ ... }'`).

Also: In Grammar 1, `action` used `expression=XBlockExpression` *outside* braces? No - Grammar 1 used `expression=XBlockExpression` *inside* the body. Actually:

Grammar 1:
```dsl
IsisAction:
  => (...) (returnType=JvmTypeReference)? name=ID '(' parameters+=... ')'
     expression=XBlockExpression;  //  this means: action has *one* block expression as body
```

Grammar 2:
```dsl
IsisAction:
  => (...) type=JvmTypeReference name=ID '{'
     (features+=IsisActionFeature | parameters+=IsisActionParameter | events+=IsisEvent)*
     '}';  
```

 This is **incompatible**:  
- Grammar 1 had a *single* expression as action body.  
- Grammar 2 has *no explicit action body expression*! Instead, body is composed of:
  - `parameters`  
  - `features` (e.g., `Body='body' expression=XBlockExpression`)  
  - `events`

-> So in Grammar 2, the *body expression* must now be expressed as an `IsisActionFeature` of type `Body`:

```dsl
IsisActionFeature:
  annotations+=XAnnotation* type=IsisActionFeatureType expression=XBlockExpression;

enum IsisActionFeatureType:
  Body='body' | Hide='hide' | Disable='disable' | Validate='validate';
```

 So we must replace the action block `{ ... }` with:
```dsl
{ 
  body { ... }
  parameter ...
}
```

But wait - in Instance 1, the actions *already* have a block body.

Let's re-express:

Original:
```dsl
action updateName(String newName) {
  setName(newName)
  this
}
```

-> Must become:
```dsl
action SimpleObject updateName {
  body {
    setName(newName)
    this
  }
  parameter String newName
}
```

Similarly for others.

Also: `default0UpdateName()` had no parameters, so:
```dsl
action String default0UpdateName {
  body { getName() }
}
```

And `validateUpdateName(String name)`:
```dsl
action TranslatableString validateUpdateName {
  body {
    if (name.contains("!")) TranslatableString.tr("Exclamation mark is not allowed")
    else null
  }
  parameter String name
}
```

 This preserves semantics *while conforming to Grammar 2*.

Also: The annotations `@Action(...)`, `@ActionLayout(...)`, etc., are part of `XAnnotation` and go *before* `'action'`. So we keep them.

But - in Instance 1, `@Action(domainEvent = UpdateNameDomainEvent)` is now redundant since we removed the top-level event - but it's still syntactically allowed and may be needed for runtime (annotations may still be supported). So we **preserve** it.

But wait - in Instance 1, `event` appears *after* `property String name`, on its own line. Grammar 1 allowed it as a top-level member. Grammar 2 does not. So removal is mandatory.

```dsl
package domainapp.dom.modules.simple

import javax.jdo.annotations.Column
import javax.jdo.annotations.DatastoreIdentity
import javax.jdo.annotations.IdGeneratorStrategy
import javax.jdo.annotations.IdentityType
import javax.jdo.annotations.PersistenceCapable
import javax.jdo.annotations.Queries
import javax.jdo.annotations.Query
import javax.jdo.annotations.Unique
import javax.jdo.annotations.Version
import javax.jdo.annotations.VersionStrategy
import org.apache.isis.applib.annotation.Action
import org.apache.isis.applib.annotation.ActionLayout
import org.apache.isis.applib.annotation.BookmarkPolicy
import org.apache.isis.applib.annotation.DomainObject
import org.apache.isis.applib.annotation.DomainObjectLayout
import org.apache.isis.applib.annotation.DomainServiceLayout
import org.apache.isis.applib.annotation.Editing
import org.apache.isis.applib.annotation.MemberOrder
import org.apache.isis.applib.annotation.Parameter
import org.apache.isis.applib.annotation.ParameterLayout
import org.apache.isis.applib.annotation.Property
import org.apache.isis.applib.annotation.SemanticsOf
import org.apache.isis.applib.annotation.Title
import org.apache.isis.applib.query.QueryDefault
import org.apache.isis.applib.services.i18n.TranslatableString

@PersistenceCapable(identityType=IdentityType.DATASTORE)
@DatastoreIdentity(strategy=IdGeneratorStrategy.IDENTITY, column="id")
@Version(strategy=VersionStrategy.VERSION_NUMBER, column="version")
@Queries(#[
	@Query(name = "find", language = "JDOQL",
		value = "SELECT FROM domainapp.dom.modules.simple.SimpleObject"),
	@Query(name = "findByName", language = "JDOQL",
		value = "SELECT FROM domainapp.dom.modules.simple.SimpleObject WHERE name.indexOf(:name) >= 0")
])
@Unique(name="SimpleObject_name_UNQ", members = #["name"])
@DomainObject(objectType = "SIMPLE")
@DomainObjectLayout(bookmarking = BookmarkPolicy.AS_ROOT)
entity SimpleObject {

	@Column(allowsNull="false", length = 40)
	@Title(sequence="1")
	@Property(editing = Editing.DISABLED)
	property String name

	@Action(domainEvent = UpdateNameDomainEvent)
	action SimpleObject updateName {
		body {
			setName(newName)
			this
		}
		parameter String newName
	}

	action String default0UpdateName {
		body {
			getName()
		}
	}

	action TranslatableString validateUpdateName {
		body {
			if (name.contains("!"))
				TranslatableString.tr("Exclamation mark is not allowed")
			else null
		}
		parameter String name
	}

	title {
		TranslatableString.tr("Object: {name}", "name", name)
	}

}
```
