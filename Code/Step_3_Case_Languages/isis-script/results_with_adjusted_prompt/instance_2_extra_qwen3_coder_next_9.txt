#### Grammar 2 expectations:
- `action` now requires: `action <type> <name> '{' ... }'`
- Parameters are now inside: `parameter <type> { ... }?`
- Return type is *not optional* -> must be added explicitly.

But note: In Apache Isis, actions like `default0UpdateName()` and `validateUpdateName(...)` are *auxiliary actions*:
- `default0UpdateName()` -> `default` for first parameter of `updateName`
- `validateUpdateName(...)` -> validation hook for `updateName`

However, per grammar:  
- `IsisActionParameter` syntax:  
  ```xtext
  IsisActionParameter:
    => (annotations+=XAnnotation* 'parameter') type=FullJvmFormalParameter ('{' features+=IsisActionParameterFeature* '}')?;
  ```
-> So `parameter` must be preceded by keyword `'parameter'` - and it's *inside* `{ ... }`.

Therefore, the full action becomes:

```dsl
action String updateName '{'
  events+=IsisEvent
  parameters+=IsisActionParameter
  features+=IsisActionFeature  // body
'}';
```

So for `updateName`, we need:
- Return type: `String`
- `events+=IsisEvent` -> `event UpdateNameDomainEvent`
- `parameters+=IsisActionParameter` -> `parameter String newName { ... }`
- `features+=IsisActionFeature` -> `body` -> `expression=XBlockExpression` -> but note: `IsisActionFeature` uses enum: `{ Body='body' | Hide='hide' | Disable='disable' | Validate='validate' }`  
  -> So the method body is now expressed via `body { ... }`, **not** as direct `XBlockExpression` outside.

 Critical: In Grammar 2, `IsisAction` no longer has `expression=XBlockExpression` - instead, it has `features+=IsisActionFeature` inside `{ ... }`, and one of them is `body`.

So the action must be rewritten as:

```dsl
action String updateName '{'
  events+=IsisEvent
  parameters+=IsisActionParameter
  features+=IsisActionFeature  // body
'}';
```

Where:
- `IsisActionFeature`: `type=Body` -> body
- So: `body { ... }` for the body.
