## 1) Differences between Grammar 1 and Grammar 2 (what affects instances)

1. **Domainmodel changed**
   - **Grammar 1:** only `(elements+=Component)*`
   - **Grammar 2:** allows first `(elements+=AttributeList)*` and then `(elements+=Component)*`
   - Impact: An instance may optionally define `attribute_list` blocks at top level. (But optional → don’t add unless needed.)

2. **New top-level rule: `AttributeList`**
   - Syntax:
     ```
     attribute_list name=ID {
        (attributes+=Attribute)*
     }
     ```
   - Impact: Optional. Only needed if components reference them.

3. **Component body now optionally references attribute lists**
   - **Grammar 1 component body:** `{ (attributes+=Attribute)* }`
   - **Grammar 2 component body:** `{ (attributes+=Attribute)* ('+' attributeLists+=ID)* }`
   - Impact: You *may* add lines like `+ commonAttributes` at the end of a widget body, but it’s optional → do **not** add.

4. **Component header adds an optional flag**
   - New optional keyword: `is_responsive` (stored in `isReponsive`)
   - Impact: Optional → do **not** add.

5. **Attributetype extended**
   - Added: `Float`, `javax.el.ValueExpression`, `Drawing`, `java.lang.Object`, `ScrollSpyEventListener`, `Node`
   - Existing `Map<...>` formatting changed slightly:
     - **Grammar 1:** `'Map<' QualifiedName ', ' QualifiedName '>'` (requires comma+space)
     - **Grammar 2:** `'Map<' QualifiedName ',' QualifiedName '>'` (requires comma with **no space** after it)
   - Impact: Only matters if the instance contains `Map<...,...>` types.

## 2) What must change in Instance 1 to conform to Grammar 2

Because rules/flags are only *added* and are optional, **almost all of Instance 1 already conforms to Grammar 2 unchanged**.

The only potential non-conformance is the `Map<...,...>` **type literal** spacing:
- Grammar 2 requires: `Map<QualifiedName,QualifiedName>` (no space after comma).
- In Instance 1, I see at least:
  ```
  dataTable-properties Map<DataTablePropertyType, Object> ...
  ```
  This has a space after the comma, so it must become:
  ```
  dataTable-properties Map<DataTablePropertyType,Object> ...
  ```

Per your rule (2), I do not instantiate new optional constructs (`attribute_list`, `+ ...`, `is_responsive`, new types).
